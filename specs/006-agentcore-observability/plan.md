# Implementation Plan: Enable AgentCore Observability

**Branch**: `006-agentcore-observability` | **Date**: 2026-03-10 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/006-agentcore-observability/spec.md`

## Summary

Enable comprehensive observability for the AgentCore-deployed Strands agent across three layers: (1) Runtime tracing and application log delivery, (2) Identity tracing and log delivery, and (3) agent-level OTEL instrumentation via `strands-agents[otel]`. CloudFormation handles Transaction Search setup (`AWS::XRay::TransactionSearchConfig` + `AWS::Logs::ResourcePolicy`), IAM, log groups, and container dependency updates. AWS CLI handles API-only toggles (Runtime/Identity tracing enable, log delivery configuration). Zero manual Console steps.

## Architecture

### Layer 1: CloudWatch Transaction Search (Account-Level)

One-time account setup. Enables OTEL span ingestion into the `aws/spans` CloudWatch Logs group with 100% sampling.

**CloudFormation** (preferred — aligns with IaC-first constraint):
1. `AWS::Logs::ResourcePolicy` — grant X-Ray service permission to write spans to CloudWatch Logs (`aws/spans` + `/aws/application-signals/data`)
2. `AWS::XRay::TransactionSearchConfig` — enable Transaction Search with `IndexingPercentage: 100`

**CLI fallback** (if CFN resources not yet supported in region):
1. `aws logs put-resource-policy` — grant X-Ray write access to CloudWatch Logs
2. `aws xray update-trace-segment-destination --destination CloudWatchLogs` — route segments to CloudWatch Logs
3. `aws xray update-indexing-rule` — set sampling percentage to 100%

### Layer 2: Runtime Observability (Per-Resource)

**Tracing**: Enable via AgentCore API/CLI (not in CloudFormation — no CFN property exists).
**Log delivery**: Configure via CloudWatch Logs Vended Logs APIs (3-step pattern):
- `put-delivery-source` — register Runtime ARN as source for `APPLICATION_LOGS` and `USAGE_LOGS`
- `put-delivery-destination` — register destination (CloudWatch Logs group ARN, type `CWL`)
- `create-delivery` — link each source to its destination

**CloudFormation additions**:
- Pre-create CloudWatch Log Groups for application and usage logs
- Expand IAM if needed for OTEL log group access

### Layer 3: Identity Observability (Per-Resource)

Same pattern as Runtime — enable tracing and configure log delivery for the WorkloadIdentity resource associated with the Runtime's JWT authorizer.

### Layer 4: Agent OTEL Instrumentation (Container)

**Single change**: Replace `strands-agents>=0.1.0` with `strands-agents[otel]>=0.1.0` in `requirements-agent.txt`. This adds Strands-specific OTEL instrumentors (agent loop, LLM call, tool call spans).

The Dockerfile already has:
- `aws-opentelemetry-distro>=0.10.1` (OTEL distro with X-Ray exporter)
- `CMD ["opentelemetry-instrument", "python", "app.py"]` (auto-instrumentation wrapper)

**Deployment**: Requires CodeBuild rebuild + Runtime update to pick up the new container image.

## Implementation Approach

### What goes in CloudFormation (`template.yaml`)

1. **Transaction Search** — `AWS::Logs::ResourcePolicy` + `AWS::XRay::TransactionSearchConfig` with 100% indexing
2. **CloudWatch Log Groups** — pre-create log groups for observability:
   - Runtime application logs group
   - Runtime usage logs group
   - Identity application logs group
3. **IAM policy expansion** (if needed) — add permissions for observability-specific log groups

### What goes in CLI scripts (`quickstart.md`)

1. **Runtime tracing enable** — AgentCore API call (no CFN property for this)
2. **Runtime log delivery** — `put-delivery-source` + `put-delivery-destination` + `create-delivery` for APPLICATION_LOGS and USAGE_LOGS
3. **Identity tracing enable** — AgentCore API call
4. **Identity log delivery** — `put-delivery-source` + `put-delivery-destination` + `create-delivery`
5. **Runtime tracing for traces** — `put-delivery-source` (logType=TRACES) + `put-delivery-destination` (type=XRAY) + `create-delivery`

### What goes in container rebuild

1. **`requirements-agent.txt`** — change `strands-agents>=0.1.0` to `strands-agents[otel]>=0.1.0`
2. **CodeBuild trigger** — rebuild container image
3. **Runtime update** — deploy new image to AgentCore Runtime
