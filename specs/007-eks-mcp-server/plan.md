# Implementation Plan: EKS MCP Server Integration

**Branch**: `007-eks-mcp-server` | **Date**: 2026-03-14 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/007-eks-mcp-server/spec.md`

## Summary

Integrate the AWS-managed EKS MCP Server into the Strands reasoning chatbot so users can query and troubleshoot their EKS clusters via natural language. Uses `mcp-proxy-for-aws` library mode for SigV4-authenticated connections. All IAM permissions managed via the existing CloudFormation template.

## Technical Context

**Language/Version**: Python 3.11+
**Primary Dependencies**: strands-agents>=0.1.0, mcp-proxy-for-aws (new), strands-agents-tools>=0.1.0
**Storage**: N/A ‚Äî stateless, no persistent data
**Testing**: pytest
**Target Platform**: Linux (AgentCore container) + local dev (WSL2)
**Project Type**: Web application (Streamlit + agent backend)
**Performance Goals**: EKS query responses within 10 seconds
**Constraints**: Read-only EKS MCP tools only; least-privilege IAM
**Scale/Scope**: Single-user demo; EKS clusters in one AWS region per deployment

## Architecture

### Integration Points

1. **`src/agent/mcp_tools.py`** (NEW) ‚Äî MCP client factory and tool loader
   - Creates `aws_iam_streamablehttp_client` factory for EKS MCP Server
   - Resolves region from env vars (EKS_MCP_REGION ‚Üí AWS_REGION ‚Üí AWS_DEFAULT_REGION)
   - Returns list of MCP tools for agent consumption
   - Handles connection errors gracefully (logs warning, returns empty tool list)

2. **`src/agent/chatbot.py`** (MODIFY) ‚Äî Add MCP tools to agent
   - Import and call MCP tool loader
   - Merge EKS MCP tools with existing Tavily tool
   - Agent created with combined tool list

3. **`src/agentcore/app.py`** (MODIFY) ‚Äî Same changes for AgentCore mode
   - Import and call MCP tool loader
   - Agent in `invoke()` entrypoint gets MCP tools alongside Tavily

4. **`infra/agentcore/template.yaml`** (MODIFY) ‚Äî IAM permissions
   - Add `EksMcpAccess` policy to `AgentExecutionRole` with eks-mcp:* read actions
   - Add `EksReadAccess` policy with underlying eks:* read actions
   - Add `EksSupportingAccess` policy with logs, cloudwatch, sts, iam read actions
   - Add `EKS_MCP_REGION` to Runtime environment variables

5. **`pyproject.toml`** (MODIFY) ‚Äî Add `mcp-proxy-for-aws` dependency
6. **`infra/agentcore/requirements-agent.txt`** (MODIFY) ‚Äî Add `mcp-proxy-for-aws` dependency
7. **`.env.example`** (MODIFY) ‚Äî Add `EKS_MCP_REGION` documentation
