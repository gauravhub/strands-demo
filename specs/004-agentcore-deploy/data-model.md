# Data Model: Deploy Strands Agent to AgentCore

**Branch**: `004-agentcore-deploy` | **Date**: 2026-03-10

## Overview

This feature introduces no persistent data store. All state is ephemeral (held in `st.session_state` or in-flight HTTP). The entities below describe runtime structures passed between components.

---

## Entity 1: `AgentCoreConfig`

Holds all configuration needed for the Streamlit app to locate and call the AgentCore Runtime endpoint.

| Field | Type | Source | Description |
|---|---|---|---|
| `runtime_arn` | `str` | `AGENTCORE_RUNTIME_ARN` env var | Full ARN of the deployed AgentCore Runtime |
| `region` | `str` | `AWS_REGION` env var | AWS region where the Runtime is deployed |
| `qualifier` | `str` | Fixed: `"DEFAULT"` | Runtime version qualifier |

**Validation rules**:
- `runtime_arn` MUST match the pattern `arn:aws:bedrock-agentcore:{region}:{account}:runtime/{id}`
- `region` MUST be a non-empty string
- Raises `EnvironmentError` at startup if any required env var is missing

**Lifecycle**: Created once at Streamlit app startup; immutable for the app's lifetime.

---

## Entity 2: `InvocationSession`

Represents an active multi-turn conversation session between a Streamlit browser tab and the AgentCore Runtime.

| Field | Type | Stored In | Description |
|---|---|---|---|
| `session_id` | `str` | `st.session_state["agentcore_session_id"]` | UUID v4 (36 chars); generated once per browser tab |
| `messages` | `list[ChatMessage]` | `st.session_state["messages"]` | Ordered list of turns in this conversation |
| `is_streaming` | `bool` | `st.session_state["is_streaming"]` | Guard flag to prevent concurrent submissions |

**Validation rules**:
- `session_id` MUST be ≥ 33 characters (AgentCore minimum); UUID v4 satisfies this
- A new `session_id` is generated on every page load (ephemeral — no persistence across browser refreshes)
- `messages` is empty on initialization

**Lifecycle**: Created by `init_session()` on first page load; destroyed on page refresh or tab close.

---

## Entity 3: `ChatMessage`

A single turn in the conversation — either a user input or an agent response.

| Field | Type | Description |
|---|---|---|
| `role` | `Literal["user", "assistant"]` | Identifies the speaker |
| `content` | `str` | Text content of the message (accumulated during streaming) |
| `reasoning` | `str` | Reasoning tokens emitted during extended thinking (assistant only) |
| `tool_calls` | `list[ToolCall]` | Tool invocations made during this turn (assistant only) |
| `error` | `str \| None` | Error message if the turn failed |

**Validation rules**: Unchanged from feature 003 — same structure, same rendering logic.

---

## Entity 4: `ToolCall`

Represents a single tool invocation within an assistant message turn.

| Field | Type | Description |
|---|---|---|
| `tool_use_id` | `str` | Unique identifier for this invocation |
| `name` | `str` | Tool name (e.g., `"tavily"`) |
| `input` | `dict` | Tool input parameters |
| `result` | `str \| None` | Tool output; `None` until result arrives |

**Validation rules**: Unchanged from feature 003.

---

## Entity 5: `AgentCoreSSEEvent`

A single Server-Sent Event emitted by the agent container over the HTTP SSE stream. This is a wire-level entity — it is never persisted.

| Field | Type | Present When | Description |
|---|---|---|---|
| `type` | `str` | Always | One of: `text`, `reasoning`, `tool_start`, `tool_result`, `error`, `done` |
| `data` | `str` | `type in {text, reasoning}` | Token content |
| `tool_use_id` | `str` | `type in {tool_start, tool_result}` | Tool correlation ID |
| `name` | `str` | `type == tool_start` | Tool name |
| `input` | `dict` | `type == tool_start` | Tool input parameters |
| `result` | `str` | `type == tool_result` | Tool output text |
| `message` | `str` | `type == error` | Error description |

**Lifecycle**: Generated by `src/agentcore/app.py` inside the container; consumed and discarded by `src/agentcore/client.py` in the Streamlit process. Never stored.

**SSE wire format**:
```
data: {"type": "text", "data": "Hello "}\n\n
data: {"type": "text", "data": "world"}\n\n
data: {"type": "done"}\n\n
```

---

## Entity 6: `InvocationRequest`

The HTTP request body sent from Streamlit to the AgentCore Runtime endpoint.

| Field | Type | Description |
|---|---|---|
| `prompt` | `str` | The user's message for this turn |

**Wire format**: `application/json` POST body: `{"prompt": "user message text"}`

**HTTP headers**:
| Header | Value |
|---|---|
| `Authorization` | `Bearer {cognito_access_token}` |
| `Content-Type` | `application/json` |
| `X-Amzn-Bedrock-AgentCore-Runtime-Session-Id` | `{session_id}` (UUID v4, 36 chars) |

## Infrastructure State (CloudFormation)

The following AWS resources are created by the CloudFormation stack. These are infrastructure entities, not application data entities.

| Resource | CFN Type | Key Attributes |
|---|---|---|
| ECR Repository | `AWS::ECR::Repository` | Holds the ARM64 agent container image |
| Agent Execution Role | `AWS::IAM::Role` | Trusted by `bedrock-agentcore.amazonaws.com`; has CloudWatch + X-Ray permissions |
| CodeBuild Project | `AWS::CodeBuild::Project` | ARM64 build environment; builds and pushes the container image to ECR |
| CodeBuild Role | `AWS::IAM::Role` | Allows CodeBuild to push to ECR and write build logs |
| Build Trigger Lambda | `AWS::Lambda::Function` | Custom resource handler; triggers CodeBuild and waits for completion during stack deployment |
| Lambda Execution Role | `AWS::IAM::Role` | Allows Lambda to start CodeBuild and report status |
| AgentCore Runtime | `AWS::BedrockAgentCore::Runtime` | The deployed agent; secured with Cognito JWT authorizer; public network mode |

**Key immutability constraint**: `AgentRuntimeArtifact.ContainerConfiguration.ContainerUri` is immutable in CFN — changing the ECR image URI requires resource replacement. For this iteration, the stack is always deployed fresh; blue/green versioning is deferred.
